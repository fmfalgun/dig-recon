> WHAT IS DIG RECON

A full DNS sweep goes beyond dig A domain.com. dig-recon queries all critical record types (A/AAAA/NS/MX/TXT/SOA/CAA/DNSKEY), resolves DMARC via CNAME chains, attempts AXFR zone transfer against each nameserver, brutes a common subdomain wordlist, performs PTR reverse-DNS on discovered IPs, and fingerprints the hosting provider. Output is structured JSON.

> WHY THIS SCRIPT

Raw dig output is one record type at a time, unstructured text, no analysis. This script combines all queries, classifies the hosting environment, detects Cloudflare edge nodes, identifies email provider from MX patterns, evaluates SPF + DMARC spoofability, and emits a complete machine-readable result with a pentester-oriented summary.

> USAGE

Basic sweep:

python3 dig-recon.py --domain target.com
# queries all DNS record types, brutes subdomains, outputs JSON summary

Save JSON output:

python3 dig-recon.py --domain target.com --output results.json
# writes full structured output to file in addition to terminal display

Skip cache (force fresh queries):

python3 dig-recon.py --domain target.com --no-cache
# bypasses cache.db — always queries live DNS resolvers

Override cache TTL:

python3 dig-recon.py --domain target.com --ttl 3600
# treat cached results older than 3600 seconds as stale

Submit to DNS Board:

python3 dig-recon.py --domain target.com --submit
# opens PR / posts result to the public DNS Board index

Reconfigure API / resolver settings:

python3 dig-recon.py --reconfigure
# re-runs first-time setup: resolver, output preferences, submit endpoint

> EMAIL SPOOFABILITY

SPF and DMARC work together. SPF alone does not prevent spoofing of the From: header visible to recipients — that requires DMARC alignment.

  • No SPF + No DMARC — fully open. Any MTA can send mail as this domain. Trivially spoofable.
  • SPF present, No DMARC — SPF validates the envelope sender but the visible From header is unprotected. Still spoofable at the header level.
  • DMARC p=none — monitoring only. Reports are sent but no enforcement action is taken. Spoofing succeeds in all mail clients.
  • DMARC p=quarantine — spoofed mail is sent to spam. Partial protection depending on recipient mail server compliance.
  • DMARC p=reject pct=100 — fully protected. Compliant MTAs reject spoofed messages outright.

Common pentest finding: domains with no DMARC record are trivially spoofable regardless of SPF configuration. dig-recon flags email_spoofable: true whenever the combination of SPF + DMARC leaves the domain open to header-From spoofing.
